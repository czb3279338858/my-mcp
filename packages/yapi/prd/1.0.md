## 1. 产品概述

### 1.1 产品背景
Yapi 是广泛使用的 API 管理平台。在 AI 辅助编程场景中，开发者需要高效获取 Yapi 中的接口定义、项目结构和字段信息。通过构建符合 MCP 标准的服务器，可让 AI 助手直接读取 Yapi 数据，实现自然语言驱动接口查阅与代码生成。

### 1.2 产品目标
- 提供标准 MCP Server，使 AI 安全、高效地访问 Yapi 数据。
- 覆盖接口搜索、详情查询、项目/分类浏览、字段级搜索等高频场景。
- 兼容主流 MCP 客户端及 Yapi 私有化部署版本，同时支持较旧版本认证方式。
- 支持 Node.js 18+（MCP SDK 及内置 fetch 要求）。

### 1.3 用户画像
- **前端/后端开发者**：在 IDE 中通过 AI 查询接口参数、响应结构及字段定义。
- **测试工程师**：定位包含特定请求/响应字段或备注的接口，用于测试用例设计。
- **技术负责人**：全局浏览项目接口树，管理多项目 Token 配置。

## 2. 产品范围

### 2.1 V1.0 包含功能
1. **项目浏览**：查看有权限的 Yapi 项目列表及基本信息。
2. **分类/接口树**：查看项目下的接口分类与接口列表。
3. **接口详情查询**：根据 ID 或路径+方法获取接口完整定义（不含 Mock 规则）。
4. **全局关键字搜索**：按关键词（路径、名称、描述）搜索接口。
5. **按字段搜索接口**：根据请求/响应字段名称或备注搜索包含该字段的接口（依赖本地索引）。
6. **本地索引构建**：提供 `build_index` 工具，全量/增量构建接口字段索引，持久化为本地 JSON 文件（路径可配置），支持搜索前自动增量检查（cooldown 可配置，默认 5 分钟）。
7. **单实例支持**（V1）：配置单个 Yapi 服务器，多实例留待后续版本。
8. **多鉴权方式**：支持项目 Token 数组（绑定项目ID）、账号密码+Cookie 等。
9. **配置辅助工具**：提供一键获取所有项目 Token 的工具，输出配置片段。

### 2.2 暂不包含
- 获取接口 Mock 规则与示例数据。
- 创建/修改接口等写入操作。
- 测试集合、统计、变更推送等功能。

## 4. 功能需求

### 4.1 MCP 工具（Tools）定义

#### 4.1.1 `list_projects`
- **描述**：获取当前认证用户有权访问的项目列表。
- **输入**：无（可选分页参数 `limit`、`offset`）。
- **输出**：项目数组，包含项目 ID、名称、描述、拥有者、接口数量等。

#### 4.1.2 `get_interface_tree`
- **描述**：获取指定项目下的接口分类树及接口概览。
- **输入**：`project_id`（必填）。
- **输出**：嵌套分类节点，每个节点含分类名称、接口列表（接口 ID、标题、路径、方法）。

#### 4.1.3 `get_interface_detail`
- **描述**：获取指定接口的完整定义（不含 Mock 规则）。
- **输入**：`interface_id`（必填），或 `project_id` + `path` + `method`。
- **输出**：
  - 基本信息：标题、路径、方法、状态、标签、更新时间
  - 请求参数：Query、Header、Body 字段名、类型、必填、说明、示例
  - 响应参数：状态码、字段结构、说明、示例
  - 备注/创建人/更新人

#### 4.1.4 `search_interfaces`
- **描述**：通过关键词在项目中搜索接口。
- **输入**：`keyword`（必填，匹配路径/名称/描述）、`project_id`（可选）、分页参数。
- **输出**：匹配接口列表，包含接口 ID、标题、路径、方法、项目名称。

#### 4.1.5 `search_interfaces_by_field`
- **描述**：根据请求参数或响应中的字段名称、备注，搜索包含该字段的接口。
- **输入**：
  - `field_name`（必填）：字段名或备注关键词。
  - `search_in`（可选）：`request`（搜索请求参数字段及备注）、`response`（搜索响应字段及备注）、`both`（默认，搜索两者）。
  - `project_id`（可选）：限定项目。
  - 分页参数。
- **输出**：匹配接口列表，每个接口附带匹配的字段详情（字段名、位置、匹配的备注等）。
- **实现说明**：
  - Yapi 无原生字段搜索接口，依赖本地持久化索引进行搜索。
  - 索引不存在时返回提示信息，建议先调用 `build_index`。
  - 索引存在时，若距上次检查超过 `refreshCooldown`（默认 5 分钟，可配置），自动执行增量检查：通过对比每个接口的 `up_time` 判断是否有变更，仅拉取变更接口的详情更新索引后再执行搜索。
  - 若在 cooldown 内则直接使用现有索引搜索，零延迟。

#### 4.1.7 `build_index`
- **描述**：构建或更新本地接口字段索引，用于 `search_interfaces_by_field` 的数据源。
- **输入**：
  - `mode`（可选）：`full`（全量重建）、`incremental`（增量更新，默认）。
- **输出**：构建结果摘要，包含项目数、接口数、新增/更新/删除数量、耗时。
- **实现说明**：
  - 全量模式：遍历所有项目 → 分类 → 接口详情，提取字段信息写入索引文件。
  - 增量模式：获取各项目接口列表（轻量），对比每个接口的 `up_time` 与索引记录，仅重新拉取有变更的接口详情。
  - 索引持久化为本地 JSON 文件，路径通过配置项 `indexPath` 指定（默认 `~/.yapi-mcp/cache/index.json`）。
  - 索引文件跨会话、跨重启复用。

#### 4.1.6 `get_project_tokens`
- **描述**：帮助用户获取所有有权限项目的 Token，生成可直接粘贴进 MCP 配置的 JSON 片段。
- **使用场景**：用户在配置文件中填写了账号密码或基础 Cookie 后，调用此工具自动遍历项目并提取 Token，免除手动去每个项目页面复制 Token 的步骤。
- **输入**：无（依赖当前实例的认证凭据，如已配置的账号密码/Cookie）。
- **输出**：一个 JSON 对象，包含 `projectTokens` 数组，每项有 `project_id`、`project_name`、`token`。同时输出一段配置示例，提示用户复制到配置文件中。
- **安全要求**：返回的 Token 不应记录在日志中，仅以文本形式呈现给用户复制。

### 4.2 配置与鉴权

#### 4.2.1 服务器配置结构（V1 单实例）
```json
{
  "baseUrl": "https://yapi.company.com",
  "auth": {
    "type": "projectTokens",
    "tokens": [
      { "projectId": 11, "token": "${PROD_TOKEN_11}" },
      { "projectId": 12, "token": "${PROD_TOKEN_12}" }
    ]
  },
  "timeout": 10000,
  "allowHttp": false,
  "indexPath": "~/.yapi-mcp/cache/index.json",
  "refreshCooldown": 300000
}
